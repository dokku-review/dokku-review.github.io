dokku·review

install ci

Heroku-style review apps, on a server you own.

→ →

Each PR lands at pr-123.review.example.com. Wildcard DNS (*.your-domain) must already point at the server — Let's Encrypt needs it to issue certificates.

Used for expiry reminders only — never verified. Defaults to admin@<domain>.

system-postgres provisions a fresh Postgres database for each review app and injects URL env vars at deploy time. DATABASE_URL is always provisioned — toggle the rest if your app uses them.

  • ▸ DATABASE_URL — primary database Required
  • ▸ CACHE_DATABASE_URL — Solid Cache
  • ▸ QUEUE_DATABASE_URL — Solid Queue
  • ▸ CABLE_DATABASE_URL — Solid Cable

Space-separated env var names. Each gets its own database on the review server.

SSH to your server as a sudo user, then paste:

$
Before you run this
  • The server must be Ubuntu 24.04 with a public IP.
  • Wildcard DNS (*.your-domain A record) must resolve to the server — otherwise Let's Encrypt fails and the apps run on HTTP only.
  • Expect 5–10 minutes of install output.
  • The script prints generated secrets at the end. Save them.
